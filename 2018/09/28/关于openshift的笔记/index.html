关于openshift的笔记

Posted on 2018-09-28 | Edited on 2019-01-07

记录我在起openshift的时候遇到问题解决的命令和一些简单的笔记

1.oc 集中了操作
2.部署出错docker images ,docker ps 看看,不行就重启docker 试一试
3.分清楚 project,app,node,pod区分.
4.创建的项目使用template部署填写的域名,该域名会由前面创建好的router来捕捉到对应的pod中.
5.oc rsh 可以进入pod内部 又一个linux
6.pod包含一个和多个(docker),简单理解就相当于你的linux跑多个docker
7.应用对于一个或多个pod,使用Service,使他们做到ip地址相对恒定. 该ip即便在你的主机上也ping不通,但是可以访问.
8.oc expose svc 创建路由
9.oc rsync 用于同步 可以用于备份oc rsync docker-registry-1-p2crc:/registry .
10.

11.可以用来调试的命令
journalctl -fu docker (查看日志可以啊)
oc get pod –all-namespaces
curl https://localhost:8443 -k
12.
oc delete po –all
oc rollout latest dc/router
oc get event
13.对s2i的测试 反正就先 docker build 再 run 再 s2i build 然后docker run
自定义镜像 要在oc edit is/{name} -n openshift 添加tages ，页面才能看见。
14.oc export bc,dc,svc,is -o json –as-template=’***’ > /test.json
导出 bc,dc,svc,is(这四个就是全部了)
然后 oc create -f 这个文件 页面就添加了模板

应该就行了 可以用命令 进行builds相关操作 最后重新build
